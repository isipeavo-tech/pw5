Предметная область

Онлайн-кинотеатр - система для управления каталогом фильмов, пользователями, подписками, историей просмотров и отзывами.

Сущности:

- CinemaUser (пользователь) - id, имя, email
- Movie (фильм) - id, название, жанр, длительность (мин), год выпуска
- Subscription (подписка) - id, пользователь (FK), тип, цена, даты начала/окончания
- ViewHistory (история просмотров) - id, пользователь (FK), фильм (FK), дата просмотра, прогресс (мин)
- Review (отзыв) - id, пользователь (FK), фильм (FK), оценка (1-10), комментарий

Связи:
- Пользователь и Подписка (1:M)
- Пользователь и История просмотров (1:M)
- Пользователь и Отзыв (1:M)
- Фильм и История просмотров (1:M)
- Фильм и Отзыв (1:M)

Архитектура и структура папок

Проект разделен на модули по слоям:

```
mmm/
├── pubspec.yaml
├── bin/
│   └── mmm.dart                  # точка входа (main)
├── lib/
│   ├── online_cinema.dart        # публичный экспорт
│   └── src/
│       ├── domain/               # модели и валидация
│       │   ├── models/           # сущности
│       │   │   ├── identity.dart
│       │   │   ├── cinema_user.dart
│       │   │   ├── movie.dart
│       │   │   ├── subscription.dart
│       │   │   ├── view_history.dart
│       │   │   └── review.dart
│       │   └── validators/       # чистые функции валидации
│       │       ├── text_validator.dart
│       │       ├── number_validator.dart
│       │       └── date_validator.dart
│       ├── data/                 # SQLite и CRUD-репозитории
│       │   ├── database.dart     # открытие БД, создание таблиц
│       │   └── repositories/
│       │       ├── user_repository.dart
│       │       ├── movie_repository.dart
│       │       ├── subscription_repository.dart
│       │       ├── view_history_repository.dart
│       │       └── review_repository.dart
│       └── cli/                  # меню и ввод/вывод
│           ├── menu.dart
│           └── input_helper.dart
└── test/
    └── cinema_test.dart          # тесты
```

Что вынесено в каждый слой и почему

domain/models/ - описывает данные и преобразования (toMap/fromMap). Не зависит от SQL и консоли. Чистая модель предметной области.

domain/validators/ - чистые функции-валидаторы (проверка пустой строки, числа больше 0, даты). Могут использоваться и в CLI, и в тестах, и в будущем веб-интерфейсе без изменений.

data/database.dart - отвечает за подключение к SQLite, создание таблиц и закрытие соединения. Изолирует инфраструктурный код.

data/repositories/ - отдельные классы для CRUD-операций каждой сущности. Каждый репозиторий отвечает только за свою таблицу.

cli/ - отвечает за взаимодействие с пользователем (stdin/stdout) и навигацию по меню. Содержит ввод с валидацией через функции из domain/validators/.

bin/ - только точка входа: создание экземпляра AppDatabase, запуск меню, закрытие ресурсов.

Такое разделение упрощает поддержку, тестирование и замену инфраструктуры (например, SQLite на другой способ хранения или консоль на веб-интерфейс).

Валидация

Реализованы проверки (чистые функции в domain/validators/):

1. Обязательное текстовое поле - значение не пустое после trim()
2. Числовое поле больше 0 - цена, длительность, прогресс
3. Число в диапазоне - оценка (1-10), год выпуска (1895-2026)
4. Дата/время - корректный парсинг через DateTime.parse()
5. ID только из цифр - проверка, что ID содержит только цифры
6. Длина текста - проверка длины строки в заданном диапазоне: имя (3-50), название (2-100), жанр (2-50), комментарий (1-500)
7. Email - проверка наличия @ и . в email

Каждая функция возвращает null при успехе или строку с ошибкой при неудаче.

Тесты

В проекте реализованы тесты (test/cinema_test.dart):

1. Модели: toMap/fromMap для всех 5 сущностей (5 тестов)
2. Валидация: 16 тестов на все валидаторы (пустая строка, число больше 0, диапазон, дата, numeric ID, длина, email)
3. Репозитории SQLite: CRUD для каждой сущности, включая update (8 тестов: вставка, чтение, обновление, удаление)
4. dumpAllTables: проверка вывода всех таблиц БД (пустых и с данными) (2 теста)
